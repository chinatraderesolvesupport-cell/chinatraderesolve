ChinaTradeResolve Document AI v3.5.8 — итоговая проверка
Дата проверки: 2026-07-22

Результат:
- 90 автоматических тестов пройдены;
- Python-код компилируется;
- структура проекта находится прямо в корне архива;
- секретные ключи и .env отсутствуют;
- безопасный .env.example включён;
- версия приложения и X-App-Version равны 3.5.8.

Подтверждённая проблема v3.5.7:
Проверка PDF и особенно декодирование/перекодирование изображений выполнялись синхронно внутри async-маршрута. Обработка одного 25–30-мегапиксельного изображения занимала около секунды CPU, а пакет из нескольких файлов мог на заметное время блокировать основной event loop одного Uvicorn worker. В этот период задерживались /health, форма и запросы других посетителей.

Исправление v3.5.8:
- CPU-затратная обработка документов перенесена в worker thread через asyncio.to_thread;
- event loop остаётся отзывчивым во время декодирования и перекодирования изображений;
- число одновременных тяжёлых обработок ограничено двумя на web-процесс;
- файлы одного пакета обрабатываются последовательно для предсказуемого расхода памяти;
- добавлены регрессионные тесты отзывчивости event loop и ограничения конкурентной обработки;
- восстановлен безопасный .env.example без секретов и включённых внешних интеграций.

Ограничение проверки:
Chromium в этой среде блокирует доступ к localhost с ERR_BLOCKED_BY_ADMINISTRATOR, поэтому полноценный клик-тест браузером не засчитан. Серверные маршруты проверены FastAPI TestClient, JavaScript проверяется отдельно перед упаковкой.
