# Установка ChinaTradeResolve v3.6.0 на Render

## 1. Сделайте резервную копию

Скачайте текущий репозиторий или сохраните старый ZIP. Это позволит быстро вернуться к прежней версии.

## 2. Замените файлы в GitHub

Распакуйте архив `ChinaTradeResolve_Document_AI_v3.6.0.zip`.

В GitHub загрузите **содержимое папки** `ChinaTradeResolve_Document_AI_v3.6.0`, а не саму внешнюю папку. Файлы `app`, `tests`, `requirements.txt`, `Dockerfile` и остальные должны находиться в корне репозитория, как раньше.

Не загружайте `.env`, пароли, ключ OpenAI, seed-фразы и приватные ключи.

## 3. Проверьте переменные Render

Сохраните уже работающие значения:

- `DATABASE_URL`;
- `APP_SECRET`;
- `ADMIN_TOKEN`;
- `PUBLIC_BASE_URL=https://chinatraderesolve.onrender.com`;
- настройки электронной почты;
- настройки ИИ, когда помощник должен работать.

Для безопасного запуска оставьте:

```env
ENABLE_VOLUNTARY_SUPPORT=false
```

Для включения нового анализа документов добавьте:

```env
ENABLE_DOCUMENT_ANALYSIS=true
OPENAI_DOCUMENT_MODEL=модель_с_поддержкой_изображений_и_PDF
DOCUMENT_ANALYSIS_MAX_OUTPUT_TOKENS=6000
DOCUMENT_ANALYSIS_TIMEOUT_SECONDS=90
APPLICATION_TRIAGE_TIMEOUT_SECONDS=8
MAINTENANCE_INTERVAL_SECONDS=60
RETENTION_CHECK_INTERVAL_SECONDS=86400
```

Также должны оставаться непустыми `OPENAI_API_KEY` и `DATABASE_URL`. Если `OPENAI_DOCUMENT_MODEL` пуст, используется `OPENAI_MODEL`.

Старые `BTC_ADDRESS`, `ETH_ADDRESS`, `USDT_TRC20_ADDRESS`, `SOL_ADDRESS` и `SUPPORT_URL` можно удалить. Даже если они останутся, при `ENABLE_VOLUNTARY_SUPPORT=false` публичная страница и QR-коды не откроются.


## Прокси-заголовки и ограничения запросов

На Render дополнительная настройка не нужна: платформа сама задаёт `RENDER=true`, и приложение использует первый корректный адрес из `X-Forwarded-For` как IP посетителя. Для запуска за другим доверенным reverse proxy можно явно задать `TRUST_PROXY_HEADERS=true`. Не включайте эту переменную при прямом публичном запуске без доверенного прокси.

## Почтовая очередь и автоматическая очистка

Версия 3.6.0 раз в минуту проверяет отложенные письма и повторяет временно неудачные отправки не более пяти раз. Закрытые дела проверяются на истечение срока хранения ежедневно. Срок отсчитывается от последнего обновления/закрытия дела, поэтому давно созданное, но только что закрытое дело не удалится преждевременно. Отдельный Cron Job для базового режима не требуется.

В v3.6.0 уведомления о создании и закрытии дела попадают в почтовую очередь в той же транзакции, что и само изменение дела. Worker арендует по одному письму непосредственно перед отправкой, поэтому длительная очередь не создаёт истёкшие заранее аренды и повторные отправки при перекрывающемся деплое.


## Обработка больших изображений

Версия 3.6.0 выполняет проверку PDF и перекодирование изображений вне основного ASGI event loop. Поэтому загрузка крупных скриншотов не блокирует `/health`, форму и запросы других посетителей. Одновременно в одном web-процессе выполняются не более двух CPU-затратных обработок документов, чтобы ограничить пиковое потребление памяти Render. Проверить значение можно в `/health`: `document_processing_workers: 2`.

## 4. Разверните новую версию

После загрузки файлов создайте commit. Render обычно начнёт новое развёртывание автоматически. Дождитесь статуса `Live` в панели Render.

## 5. Проверьте результат

Откройте:

- `https://chinatraderesolve.onrender.com/?lang=ru`;
- `https://chinatraderesolve.onrender.com/health`;
- форму на компьютере и телефоне.

В `/health` должны быть нужные значения:

```json
"support_enabled": false,
"document_analysis_enabled": true
```

Адрес `/support` должен возвращать страницу 404, пока поддержка отключена.

Проверьте первый экран, раздел «Результат», блок «О проекте» и тестовую заявку с отдельным тестовым email. После успешной отправки должен появиться номер вида `CTR-2026-...` и кнопка просмотра статуса.

## 6. Что не нужно менять

- база существующих заявок сохраняется, если `DATABASE_URL` остаётся прежним;
- ключ OpenAI не нужно вставлять в код;
- собственный домен можно подключить позднее без переделки приложения.


## 7. Проверка документов

1. Отправьте тестовую заявку с согласием на ИИ.
2. Откройте закрытую ссылку статуса.
3. Загрузите тестовый PNG или PDF без реальных персональных данных.
4. Нажмите кнопку ИИ-анализа и дождитесь отчёта.
5. Проверьте тот же файл и отчёт в `/admin`.

Подробности находятся в `DOCUMENT_AI_SETUP_RU.md`.

## Обязательная защита администратора

Задайте разные случайные значения `ADMIN_TOKEN` и `APP_SECRET` длиной не менее 32 символов каждое. Короткие значения и известные заглушки считаются небезопасными. Если хотя бы один из секретов — `ADMIN_TOKEN` или `APP_SECRET` — отсутствует, слишком короткий или оставлен значением-заглушкой, вход администратора полностью блокируется. Публичная форма при этом продолжает работать.

Анализ документов запускается в фоне внутри веб-процесса. После нажатия страница сразу возвращается, показывает статус обработки и автоматически обновляется. Пока анализ идёт, загрузка и удаление файлов блокируются, а повторный запуск не создаёт второй запрос к OpenAI. Каждый запуск имеет уникальный внутренний токен: поздний ответ старой задачи не может перезаписать новый анализ. Во время перекрывающегося zero-downtime деплоя свежая задача не помечается ошибкой; неудачной становится только задача, которая действительно остаётся в состоянии обработки дольше безопасного таймаута.


## Адрес сайта

`PUBLIC_BASE_URL` можно оставить пустым при использовании стандартного домена Render: версия 3.6.0 автоматически берёт HTTPS-адрес из `RENDER_EXTERNAL_URL`. Для собственного домена задайте `PUBLIC_BASE_URL` явно.


## Безопасное использование `.env.example`

В версии 3.6.0 секреты, публичный URL и SMTP оставлены пустыми. Не импортируйте вымышленные значения-заглушки. На Render задайте `ADMIN_TOKEN` и `APP_SECRET` вручную; `PUBLIC_BASE_URL` для стандартного домена не требуется.

## Проверка после обновления до v3.6.0

После статуса `Live` откройте `/health`. Должно отображаться `"version": "3.6.0"`. Затем проверьте одно тестовое дело:

1. загрузите PNG и небольшой незашифрованный PDF без активного содержимого;
2. запустите анализ один раз;
3. убедитесь, что во время обработки кнопки добавления и удаления документов недоступны;
4. обновите страницу и дождитесь готового отчёта;
5. проверьте журнал в админке: должен быть один запуск анализа, а не два.

Административные формы используют CSRF-защиту автоматически; отдельные переменные Render для неё не требуются.

## Ограничение тела запросов

Версия 3.6.0 отклоняет до разбора формы:
- обычные POST/PUT/PATCH запросы свыше 1 МБ;
- запрос загрузки документов свыше 50 МБ с учётом multipart-обёртки.

Это не требует отдельной переменной Render. Проверить активную конфигурацию можно через `/health`: `standard_request_limit_mb` и `document_upload_request_limit_mb`.
