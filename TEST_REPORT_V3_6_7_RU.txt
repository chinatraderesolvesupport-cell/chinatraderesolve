ОТЧЁТ ПРОВЕРКИ CHINATRADERESOLVE v3.6.7
Дата проверки: 22 июля 2026 года

ИТОГ
Сборка готова к развёртыванию на Render. Исправлена подтверждённая ошибка проверки PDF: стандартные структуры встроенных и связанных файлов больше не проходят фильтр загрузки.

ПОДТВЕРЖДЁННАЯ ПРОБЛЕМА v3.6.6
Проверка искала PDF-имя /EmbeddedFile в единственном числе. Стандартное дерево вложений использует /EmbeddedFiles, а вложения также описываются через /Filespec, /EF и /AF. Минимальные PDF с этими конструкциями успешно проходили старую функцию проверки.

ИСПРАВЛЕНО В v3.6.7
- блокируются /EmbeddedFile и /EmbeddedFiles;
- блокируются /Filespec, /EF, /AF и /AFRelationship;
- блокируются /Collection и /GoToE, связанные с PDF-портфолио и переходами к вложениям;
- PDF name escapes декодируются до проверки, поэтому /Embedded#46iles также распознаётся;
- безопасные PDF без активного или встроенного содержимого продолжают приниматься.

АВТОМАТИЧЕСКАЯ ПРОВЕРКА
- 106 тестов из 106 пройдены;
- Python-модули app, scripts и tests компилируются;
- регрессионная проверка охватывает JavaScript, шифрование, стандартное дерево вложений, file specification, associated files и обфусцированное имя вложения;
- после упаковки полный набор тестов повторно запускается по распакованному ZIP.

РУЧНАЯ ПРОВЕРКА ПРИЛОЖЕНИЯ
- /health возвращает version 3.6.7;
- публичная заявка создаётся;
- закрытая страница дела открывается;
- PNG загружается при наличии отдельного согласия на документы;
- PDF со стандартным /EmbeddedFiles отклоняется с HTTP 400 до сохранения;
- администраторский вход и очередь дел открываются;
- встроенный JavaScript отрендерированных страниц проходит node --check.

БЕЗОПАСНОСТЬ И УПАКОВКА
- проект находится прямо в корне ZIP;
- .env, реальных API-ключей, базы данных и кэша в архиве нет;
- .env.example содержит только безопасные пустые значения и настройки по умолчанию;
- Root Directory в Render должен оставаться пустым.
