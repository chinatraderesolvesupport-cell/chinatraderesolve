ChinaTradeResolve AI Assistant v3.0

Готовая многоязычная версия сайта с добровольной криптоподдержкой и первым этапом внедрения искусственного интеллекта.

Поддерживаемые языки:
- английский;
- французский;
- немецкий;
- испанский;
- русский;
- сербский.

Что реализовано:
1. Сохранены многоязычный сайт, форма, статус дела, письма, юридические страницы и страница поддержки.
2. Сохранены BTC, ETH, USDT/TRC20 и SOL с локальными QR-кодами и предупреждениями о сети.
3. Добавлен плавающий многоязычный ИИ‑помощник на главной странице.
4. Помощник объясняет работу сервиса, подсказывает категории доказательств и помогает структурировать общее описание спора.
5. Помощник не имеет доступа к базе дел, электронной почте, закрытым статусам или документам пользователя.
6. Чат не сохраняется в базе ChinaTradeResolve; для ответа передаются только последние сообщения беседы.
7. Ключ OpenAI хранится только на сервере. В браузер, HTML и JavaScript он не попадает.
8. Запросы к Responses API выполняются с параметром store: false.
9. Добавлены ограничения длины сообщений, истории, ответа и частоты запросов.
10. Добавлены предупреждения: не отправлять конфиденциальные имена, номера заказов, пароли, приватные ключи, seed-фразы и полные документы.
11. ИИ не даёт юридических гарантий, не обещает результат и направляет срочные или рискованные вопросы к человеку.
12. Обновлено уведомление об использовании ИИ на всех шести языках.

По умолчанию кнопка ИИ скрыта. Она появляется только после настройки переменных Render:

ENABLE_AI_ASSISTANT=true
OPENAI_API_KEY=ваш_секретный_ключ
OPENAI_MODEL=модель_доступная_в_вашем_проекте

Дополнительно можно задать отдельную модель:
OPENAI_ASSISTANT_MODEL=

После развёртывания откройте:
https://ваш-домен/health

В ответе должно быть:
"ai_assistant_enabled": true

Полная пошаговая инструкция находится в файле AI_ASSISTANT_SETUP_RU.md.

Проверка:
- 22 автоматических теста из 22 пройдены;
- проверена валидация чата;
- проверено скрытие виджета без ключа;
- проверен запрос к OpenAI Responses API через имитацию внешнего сервиса;
- проверено, что store=false и ключ не попадает в интерфейс;
- сохранены все прежние проверки формы, языков, статуса, безопасности и криптоподдержки.

Запуск локально:
1. Скопируйте .env.example в .env.
2. Задайте APP_SECRET и ADMIN_TOKEN.
3. Установите зависимости: python -m pip install -r requirements.txt
4. Запустите: uvicorn app.main:app --reload
5. Откройте: http://127.0.0.1:8000

Важно:
- никогда не загружайте .env или OPENAI_API_KEY в GitHub;
- не вставляйте ключ в код сайта;
- до публичного запуска проведите юридическую проверку Privacy, Terms, AI Notice и Disclaimer;
- используйте постоянную PostgreSQL-базу через DATABASE_URL на Render;
- оставьте noindex до готовности к публичной индексации.
