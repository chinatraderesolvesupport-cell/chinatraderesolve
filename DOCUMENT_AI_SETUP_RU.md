# Подключение анализа документов v3.4

## Что добавлено

После отправки заявки клиент получает закрытую ссылку дела. На этой странице он может загрузить до пяти ключевых файлов:

- PDF;
- JPG/JPEG;
- PNG;
- WebP.

Ограничения: не более 8 МБ на один файл и 25 МБ суммарно на одно дело.

Изображения перекодируются сервером, поэтому EXIF и другие встроенные метаданные удаляются. Файлы сохраняются в той же PostgreSQL-базе, что и дело, и не зависят от временного диска Render.

ИИ формирует предварительный структурированный отчёт:

- индекс готовности доказательств;
- состав и читаемость документов;
- хронологию;
- ключевые доказательства;
- возможные противоречия;
- недостающие материалы;
- факторы риска;
- рекомендуемые следующие шаги.

Индекс готовности показывает только организованность доказательств. Это не вероятность победы и не юридическое заключение.

## Переменные Render

В Render → Web Service → Environment добавьте или проверьте:

```text
ENABLE_DOCUMENT_ANALYSIS=true
OPENAI_API_KEY=ваш_секретный_API_ключ
OPENAI_DOCUMENT_MODEL=модель_с_поддержкой_изображений_и_PDF
DOCUMENT_ANALYSIS_MAX_OUTPUT_TOKENS=2200
DOCUMENT_ANALYSIS_TIMEOUT_SECONDS=90
```

Если `OPENAI_DOCUMENT_MODEL` оставить пустой, используется `OPENAI_MODEL`.

Существующие переменные `DATABASE_URL`, `APP_SECRET`, `ADMIN_TOKEN`, почтовые настройки и другие секреты сохраняйте без изменений.

## Проверка после развёртывания

Откройте:

```text
https://chinatraderesolve.onrender.com/health
```

Ожидаемое поле:

```json
{
  "document_analysis_enabled": true
}
```

Затем:

1. отправьте тестовую заявку с включённым согласием на ИИ;
2. откройте закрытую ссылку статуса;
3. загрузите один тестовый PNG или PDF без реальных персональных данных;
4. нажмите «Проанализировать документы с помощью ИИ»;
5. убедитесь, что появился отчёт;
6. войдите в `/admin` и проверьте, что файл и отчёт видны в карточке дела.

## Безопасность и приватность

- не загружайте `.env`, API-ключи и пароли в GitHub;
- не просите клиентов загружать пароли, seed-фразы, приватные ключи, полные номера банковских карт и ненужные документы личности;
- ссылка статуса является секретной ссылкой доступа к делу — её нельзя публиковать;
- загруженные документы передаются настроенному ИИ-провайдеру только после нажатия анализа и при ранее данном согласии на ИИ;
- запросы к Responses API отправляются с `store: false`;
- закрытые дела, файлы и отчёты удаляются или обезличиваются политикой хранения, по умолчанию через 90 дней;
- важные выводы всегда требуют проверки человеком.

## Практическое ограничение первой версии

Анализ запускается кнопкой и выполняется в рамках одного HTTP-запроса. Это надёжнее простой фоновой задачи для текущего Render-развёртывания, но пользователь должен оставить страницу открытой примерно до минуты. При росте нагрузки следующий этап — отдельная очередь заданий и worker.
