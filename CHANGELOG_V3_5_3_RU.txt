ChinaTradeResolve Document AI v3.5.3

1. Каждый запуск анализа документов получает уникальный run token, хранящийся в базе.
2. Поздний результат или ошибка старого worker больше не может перезаписать новый повторный анализ.
3. Восстановление зависших задач выполняется условно и атомарно: завершившийся результат не может быть ошибочно заменён статусом failed.
4. Fresh-задачи не сбрасываются во время перекрывающегося zero-downtime деплоя Render; сбрасываются только действительно устаревшие running-задачи.
5. Периодическая maintenance-задача также восстанавливает устаревшие анализы без ожидания открытия страницы пользователем.
6. Для существующих SQLite и PostgreSQL баз колонка run_token добавляется автоматически без потери данных.
7. Добавлены регрессионные тесты для позднего ответа старого запуска, перекрывающегося деплоя, гонки завершения и миграции старой базы.
8. Версия API и /health: 3.5.3.
